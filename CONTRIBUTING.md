# Contribution & Attribution Policy

To ensure project integrity, transparency, and proper citation:

1. **Pull Request Requirements**
   - All contributions must be made via pull request
   - Clear attribution and reasoning provided
   - Must pass CI, provenance, and attribution checks
   - Code review required before merge

2. **Provenance & Attribution**
   - All code usage must be logged and cited
   - Forks or derivative works must cite original repository
   - Automated workflows enforce attribution and linkage
   - See CITATION.cff for academic reference format

3. **Code Style & Quality**
   - Follow existing project structure and conventions
   - Include tests for new functionality
   - Document all public APIs and functions
   - Pass linting and type checking

4. **Compliance**
   - Follow system architecture rules and constraints
   - Pass law-enforcer validation for changes
   - Maintain deterministic behavior
   - No external I/O until governance approval

5. **Documentation**
   - Update relevant documentation
   - Add to changelog with proper versioning
   - Include examples where appropriate
   - Keep README current

See `.github/workflows/` for automated compliance checking.
